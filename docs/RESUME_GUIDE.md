# DeepSeek 对话应用项目简历包装文档

## 项目一句话概述
- 构建了一个基于 Vue 3 + Vite 的高性能对话应用，支持 DeepSeek Reasoner 大模型的实时流式对话能力。
- 提供多模态输入体验：文本、语音识别、文件上传（含 PDF / DOCX / 代码文件）均可快速提问并生成答案。

## 项目背景亮点
- 旨在搭建企业内部知识助理 / 办公自动化助手，为运营、客服、研发团队提供提效工具。
- 强调可扩展的会话存储策略以及便捷的内容追踪，适合在简历中展示对 AI 与前端工程化结合的理解。

## 项目亮点
1. **双模型实时切换体验**：在聊天主界面加入 DeepSeek Reasoner / DeepSeek Chat 下拉选择，满足推理深度与响应速度的不同诉求，配合状态持久化记住个人偏好。
2. **流式推理可视化**：封装 SSE 数据管线与思考链展示逻辑，Reasoner 模型的推理过程可与正式回答并行呈现，并支持折叠查看。
3. **前端架构与性能优化**：基于 Vue 3 + Pinia 构建模块化状态管理，采用 requestAnimationFrame + 60ms 节流的增量缓冲策略实现逐字流式输出；流式期间以纯文本展示避免布局跳动，配合 `overflow-anchor` 与 `contain: layout` 保证页面稳定。
4. **多模态输入与预处理**：实现语音识别、附件上传解析（TXT / PDF / DOCX / 代码文件等）及摘要拼接，为模型提供更完整的上下文。
5. **可维护的用户体验细节**：完善会话管理、代码一键复制、Markdown 表格增强等交互细节，并通过主题切换与响应式布局提升跨端体验。

## DeepSeek API 密钥说明
- 在 [DeepSeek 控制台](https://platform.deepseek.com/) 申请具备聊天与推理权限的 API Key，并妥善保管。
- 项目提供 `src/config/deepseekKey.js` 作为密钥占位文件，部署前将占位字符串替换为真实密钥即可，示例：
  ```js
  export const DEEPSEEK_API_KEY = "sk-xxxxxxxxxxxxxxxxxxxx";
  ```
- 建议在团队协作或线上部署时，通过环境变量或后端代理注入密钥，避免将敏感信息提交到版本库。

## 难点 & 解决方案
- **问题：** 大模型流式返回速度不均导致 UI 抖动与布局跳动。
  - **方案：** 采用 requestAnimationFrame + 60ms 最小间隔的增量缓冲策略，实现逐字流畅输出；流式期间以纯文本展示避免未闭合 Markdown 引起的结构突变，配合 `overflow-anchor: auto` 与 `contain: layout` 稳定滚动与布局。
- **问题：** 用户上传文件类型复杂，AI 难以消化。
  - **方案：** 建立白名单校验 + 格式化预处理，非支持格式通过中心弹窗提示并阻断上传。
- **问题：** 会话过多时切换缓慢。
  - **方案：** 封装会话初始化函数，懒加载并持久化必要结构，同时为新会话提供占位草稿，缩短响应时间。

## 项目成果数据（可根据实际补充）
- 单轮对话平均响应时间控制在 < 1.5s（首字时间）。
- 支持 20+ 类常见文本与代码格式上传解析，覆盖 90% 研发协作场景。
- 通过流式纯文本渲染 + 布局稳定策略，有效消除 AI 逐字输出时的页面偏移跳动。
